缓存机制

​ 从上一篇的垃圾回收机制中大家可以了解到当对象的引用计数器为0时，就会被销毁并释放内存。而实际上它并不是这么的简单粗暴，因为反复的创建和销毁会使程序的执行效率变低。python中引入了缓存机制。
​ 例如：引用计数器为0时，不会真正的销毁对象，而是将他放到一个名为free_list的链表里，之后再创建对象不会再开辟新的内存空间，而是在free_list中将之前的对象重置内部的值来使用。

  • float类型，维护的free_list链表最多可缓存100个float对象。
# 开辟内存来存储float对象，并将对象添加到refchain链表。
v1 = 3.14159
print(id(v1)) # 内存地址:2368672867048
# 引用计数器-1，如果为0则在refchain链表中移除，不削毁对象，而是将对象添加到float的free_list
del v1

# 优先去free_list中获取对象，并重置为8.888，如果free_list为空才重新开辟内存
v2 = 8.888
print(id(v2)) #内存地址:2368672867048

# 注意：引用计数器为0时，会先判断free_list中缓存个数是否满了，未满则将对象缓存，已满则直接将对象销毁。

  • int类型，不是基于free_list，而是维护一个small_list链表保存常见数据（小数据池），小数据池的范围：-5 <= value <257。即：重复使用这个范围的整数时，不会开辟新的内存。

    #去小数据池small_list中去获取38整数对象，将对象添加到refchain环状双向链表里并让引用计数器+1.
v1 = 38
print(id(v1)) # 内存地址: 140726972168176

# 去小数据池small_list中获取38整数对象，将ref_chain中的对象引用计数器+1.
v2 = 38
print(id(v2)) # 内存地址：140726972168176
#注意：在解释器启动时候-5~256就已经被加入到small_ints链表中且引用计数器初始化为1，代码中使用的值时直接去small_ints中拿来用并将引用计数器+1即可。另外，small_ints中的数据引用计数器永远不会为0（初始化时就设置为1了），所以也不会被销毁。

  • str类型，维护unicode_latin1[256]链表，内部将所有的ascii缓存起来，以后使用时就不再反复创建。

    v1 = "A"
print(id(v1)) # 内存地址：2366548050416
del v1
v1 = "A"
print(id(v1)) # 内存地址：2366548050416

 # 除此之外，Python内部还对字符串做了驻留机制，针对那么只含有字母、数字、下划线的字符串（见源码Objects/codeobject.c），如果内存中已存在则不会重新在创建而是使用原来的地址里（不会像free_list那样一直在内存存活，只有内存中有才能被重复利用）。
  v1 = "bob"
  v2 = "bobo"
  print(id(v1) == id(v2)) # 输出：True
  • list类型，维护的free_list数组对象最多可缓存80个对象。
v1 = [44,3,22]
print(id(v1)) # 内存地址: 2368673178184

del v1
v2 = ["bo","bo"]
print(id(v2)) # 内存地址：2368673178184
  • tuple类型，维护一个free_list数组且数组容量为20，数组中元素可以是链表且每个链表最多可以容纳2000个元组对象。元组的free_list数组存在数据时，是按照元组可以容纳的个数为索引找到free_list数组中的对应链表，并添加到链表里。
v1 = (1,2,3)
print( id(v1) ) # 内存地址：2366550259680
del v1  # 因元组的数量为2，所以会把这个对象缓存到free_list[3]的链表中。
v2 = ("bobo","lili","kangkang")  # 不会重新开辟内存，而是去free_list[3]对应的链表中拿到一个对象来使用。
print( id(v2) ) # 内存地址：2366550259680
  • dict类型，维护的free_list数组最多可以缓存80个对象。
v1 = {
    "name": "bobo",
    "age": "20",
     }
print(id(v1)) #内存地址：2368673137864

del v1

v2 = {
    'sex':'woman',
    'hobby': 'demo',
}
print(id(v2)) # 内存地址：2368673137864
